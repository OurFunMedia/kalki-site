# Kalki Site — AGENTS.md

## Stack

- **Next.js 16** (App Router), **React 19**, **TypeScript 5**
- **Tailwind CSS v4** (`@tailwindcss/postcss` — NOT the traditional `@tailwind base/components/utilities` approach)
- **Outstatic CMS** (Git-based; content lives in `outstatic/content/` as markdown)
- **pnpm** (lockfile: `pnpm-lock.yaml`)
- **ESLint 9** (flat config: `eslint.config.mjs`)

## Commands

```sh
npm run dev       # dev server at localhost:3000
npm run build     # production build
npm run start     # start production server
npm run lint      # eslint (flat config)
```

No test framework exists — do not look for test commands.

## CSS Conventions

- Tailwind v4 `@import "tailwindcss"` + `@theme` for custom values (no `tailwind.config.js`)
- Custom palette: `primary` (#7d5a45), `secondary` (#b59a85), `accent` (#c47b5f), `sand` (#f5f0eb), `cream` (#fbf8f5), `charcoal` (#4a3628)
- Three font variables: `--font-inter` (sans), `--font-outfit` (display), `--font-cormorant` (serif)
- Custom utility `animate-marquee` defined via `@utility` in `globals.css`

## CMS (Outstatic)

- 6 collections: `classes`, `instructors`, `workshops`, `events`, `products`, `ngo_projects`
- Content stored in `outstatic/content/` as markdown (committed to git)
- To use Outstatic locally: create `.env.local` with `OST_GITHUB_ID`, `OST_GITHUB_SECRET`, `OST_TOKEN_SECRET`
- CMS admin at `/outstatic`, API at `/api/outstatic/[...ost]`
- Several pages define hardcoded fallback data for when CMS isn't connected
- Product data is also read from `outstatic/content/metadata.json` (generated by Outstatic)

## Key Gotchas

- **No `.env.local` in repo** — Outstatic requires one for local CMS; it is gitignored
- **Contact form is non-functional** — no submit handler wired up
- **Inquiry form opens WhatsApp link** — no backend API, client-side only
- **`legacy-peer-deps=true`** in `.npmrc` — expect peer dep warnings, do not remove this flag
- **`transpilePackages: ['outstatic', 'transliteration']`** in `next.config.ts` — needed for ESM compat
- **`transliteration` path override** in `tsconfig.json` paths — uses browser bundle directly
- **No automated tests** exist — skip any test-related work
- **No CI workflows** — no GitHub Actions config
- **ESLint ignores** `.next/`, `out/`, `build/`, `next-env.d.ts` only
- Vue 4 `@theme` directive cannot coexist with traditional `tailwind.config.js` — do not add one

## Content Maintenance

- `generate_ngo_items.js` creates initial NGO project markdown files under `outstatic/content/ngo_projects/`
- Adding new CMS-backed pages requires: (1) collection registration, (2) page route, (3) `load()` + `find()` query from `outstatic/server`
- Hardcoded fallback data pattern is used across multiple pages — new CMS pages should follow the same pattern
